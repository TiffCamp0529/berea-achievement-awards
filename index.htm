📌 Purpose

This project addresses a critical gap in how Berea College manages student achievement awards: awards are siloed across departments, fund agreements and stipulations are not centrally accessible, and the annual awards cycle lacks a unified, coordinated process.

This plan covers two interconnected workstreams:

1
Award Database & Transparency DashboardA centralized, accessible platform where all campus stakeholders can see award fund agreements, stipulations, and real-time spending data.
2
Awards Cycle Process ModernizationA coordinated, campus-wide plan for running the awards cycle—from preparation through ceremony—with clear ownership, timelines, and communications.

🎯 End Goals

✓
Centralized Award DatabaseFund agreements, stipulations, and real-time spending visible to all authorized stakeholders.
✓
Unified Award Discovery WebpageA public or campus-accessible page listing all awards with descriptions, eligibility, and how to apply.
✓
Coordinated Campus Communication PlanSynchronized messaging from all departments throughout the entire awards cycle.
✓
Annual Culminating CeremonyA single, well-planned recognition event for Labor, Academic, and Service awards.

🎯 Objective

Create a transparent, centralized platform where all authorized campus stakeholders can access original fund agreements, award stipulations, and track spending or use in real time. This addresses the current situation where award details are scattered across departments with no shared visibility.

✅ What the Platform Should Include

1
Original Fund AgreementsScanned/linked copies of donor fund agreements with full stipulations for each award.
2
Award Details & EligibilityCriteria, amount, eligible class years, major restrictions, nomination vs. application, department owner.
3
Real-Time Budget TrackingAvailable fund balance, suggested number of recipients, amounts given per cycle, connected to Finance data.
4
Award HistoryPast recipients by year for audit trail and donor reporting.
5
Outlier Award FlagsIdentify awards that are off-cycle or not yet in AwardSpring so they can be brought into alignment.
6
Comprehensive Honor Societies & Clubs ListCreate and maintain an updatable directory of all campus honor societies and student organizations that confer recognition or induction — including each society's induction criteria, approximate timing, and any awards they present. This list is essential for the awards program and ceremony planning.

💡 Priority Improvement Areas

The following improvements were identified through the current process review. Each area has a brief description and recommended next step.

  • 🗂️ Improvement 1 · Comprehensive Award Database Challenge: Award fund agreements, stipulations, and spending data are siloed across Philanthropy, Finance, and individual departments with no shared visibility.
    Recommendation: Build a centralized platform (see Workstream 1) where all stakeholders can access fund documents, current balances, and recipient history. Finance provides the definitive fund list as the starting point.
  • 🌐 Improvement 2 · Award Information Webpage Challenge: Students and faculty have no single, comprehensive place to learn about all available awards—information is fragmented across the catalog, CELTS, Labor, and individual department pages.
    Recommendation: Create a campus-wide "Awards & Recognition" webpage listing every award with description, eligibility, amount (where permitted), and application instructions. Requires all areas to collaborate on content and Finance to confirm amounts.
  • 📢 Improvement 3 · Coordinated Communication Plan Challenge: Different departments send notifications to campus at different times, in different formats, creating confusion and missed opportunities for students.
    Recommendation: Develop a shared communication calendar agreed upon by all award-administering departments (AA, Labor, CELTS, Athletics). Coordinate joint announcements at key moments: pre-winter break, application launch, mid-cycle reminder, results.
  • 📥 Improvement 4 · Centralized Selection Submission Challenge: Selection committees in different departments submit their award winners to different places—some via email, some via spreadsheet, some via AwardSpring—creating a fragmented data collection problem for program compilation.
    Recommendation: Establish ONE platform or form for all departments to submit their award selections. Options: extend AwardSpring's admin features, create a shared Qualtrics form, or build a simple SharePoint submission. Decide at the Workstream 2 kickoff meeting.
  • 📦 Improvement 5 · Unified Certificate / Award Packet Challenge: Each department creates its own certificates and award packets with different formats, branding, and processes—and the President must sign each one individually.
    Recommendation: Agree on a single, consistent certificate template for all awards (customizable by award name/type). Consider pre-printing the President's signature to reduce bottleneck. Coordinate production so all packets are ready at the same time.
  • 📅 Improvement 6 · Early Ceremony Date Confirmation Challenge: The ceremony date is selected too late, leading to conflicts with other campus events and limited promotion time.
    Recommendation: Lock in the ceremony date during Summer/Fall 2026 and add it to the Academic Calendar immediately. This allows students, faculty, families, and campus event planning to coordinate well in advance.
  • 🔄 Improvement 7 · Outlier Award Cycle Alignment Challenge: Some awards (e.g., Christensen, Wood) operate on different cycles or timelines than the main awards process, creating communication complexity and student confusion.
    Recommendation: Audit all awards to identify those off-cycle. Either bring them onto the main awards cycle where possible, or create a clearly communicated sub-cycle with its own dedicated notifications and deadlines in AwardSpring.
  • 💰 Improvement 8 · Centralized Ceremony Budget Challenge: Currently there is no clear, centralized budget for the awards process (ceremony venue, reception, printing, packets). Costs are managed ad hoc across departments.
    Recommendation: Establish a shared budget line for the annual awards cycle. Finance and participating departments agree on contributions and cost-sharing model at the start of each cycle.

💬 Add to Microsoft Teams

Follow these steps to pin this dashboard as a tab in any Teams channel, so every stakeholder can access and interact with it directly inside Teams.

  • 1
    Save this file to SharePoint
    Upload berea_awards_project_plan.html to a SharePoint document library your stakeholders can access (e.g., the Academic Affairs SharePoint site).
  • 2
    Get the SharePoint file URL
    Right-click the file in SharePoint → Copy link. Or open it and copy the URL from your browser address bar.
  • 3
    Open your Teams channel
    Go to the Teams channel where you want to add the tab (e.g., Awards Modernization).
  • 4
    Add a Website tab
    Click the + button in the tab bar → search for Website → paste the SharePoint URL → name the tab (e.g., "Awards Dashboard") → click Save.
  • 5
    Share with stakeholders
    The tab is now visible to everyone in the channel. Phase statuses and award edits each person makes are saved locally in their own browser — use the 💾 Save Copy button to share a consolidated version with everyone's changes merged.
💡 Tip: For true shared real-time editing, consider embedding this in a SharePoint page with the Embed web part, or ask OCI/Heather Schill about hosting options that support a shared backend.
